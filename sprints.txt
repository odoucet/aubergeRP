Règles à donner à chaque agent
Implémente uniquement le sprint selon les specs dans docs/. 
N'ajoute rien hors périmètre. À la fin, make test doit passer sans erreur. 
Tous les tests doivent être dans tests/. Mocke les appels HTTP externes avec httpx ou pytest-asyncio. 
N'utilise pas de base de données.



L'ordre est strict jusqu'au sprint 3 (les sprints 4-7 peuvent se paralléliser partiellement si deux agents travaillent sur des fichiers disjoints, mais le sprint 3 est le prérequis dur de tout ce qui touche aux connecteurs).
