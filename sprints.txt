AubergeRP — Remaining Work
==========================
Last updated: 2026-04-27

All sprints 13–18 are complete (including the full multi-user session-token
stack: localStorage UUID, X-Session-Token header, per-session image folders,
conversation owner scoping, "Share session" button, and multi-browser SSE via
EventBus).

The items below represent all remaining unimplemented features.

--------------------------------------------------------------------------------
SPRINT 19 — Authentication
Priority: HIGH (required before any public/shared deployment)
--------------------------------------------------------------------------------
[ ] Basic access control
    - Password-protected access (HTTP Basic Auth or form-based login)
    - Or IP allowlist for trusted-network deployments
    - Admin panel protected separately from the chat UI
    - Config: app.auth_mode ("none" | "password" | "ip_allowlist")
    - Config: app.password (bcrypt hash) or app.allowed_ips (list of CIDR ranges)
[ ] Session invalidation
    - Logout endpoint: DELETE /api/session
    - Expire sessions after configurable idle timeout

--------------------------------------------------------------------------------
SPRINT 20 — AI Quality
Priority: MEDIUM
--------------------------------------------------------------------------------
[ ] Hallucination mitigation
    - Detect clearly off-topic or repetitive assistant responses
    - Configurable: retry with higher temperature or a corrective system message
    - Config: chat.hallucination_retry (bool, default false)
[ ] Enforced NSFW content filtering
    - Optional pre/post-processing layer around LLM responses
    - Config: chat.nsfw_filter ("off" | "warn" | "block")
    - Warn mode: append a disclaimer; block mode: replace response with a refusal
[ ] Manual "Generate Image" button
    - UI button in the chat input area that triggers image generation without
      relying on the LLM emitting a [IMG: ...] marker
    - POST /api/chat/{id}/generate-image with an optional prompt override

--------------------------------------------------------------------------------
SPRINT 21 — Advanced Chat Features
Priority: LOW
--------------------------------------------------------------------------------
[ ] Multi-character conversations
    - Allow more than one character to participate in a single conversation
    - Each character has its own avatar, name, and system prompt
    - Backend: conversation.characters: list[str] instead of character_id: str
    - Frontend: character tabs or turn-selector in the chat UI
[ ] Multi-model support
    - Use different connector/model for chat vs. summarization vs. classification
    - Config: chat.summarization_connector_id, chat.classification_connector_id
    - Falls back to the active text connector if not set
[ ] Automatic proactive image triggering
    - LLM decides on its own when to emit an image (not only on user request)
    - Requires prompt-engineering changes and possibly a second pass / tool call
    - Config: chat.proactive_images (bool, default false)
[ ] Quota management
    - Per-conversation token or cost limit; warn when approaching; block when exceeded
    - Config: chat.max_tokens_per_conversation, chat.max_cost_per_conversation
